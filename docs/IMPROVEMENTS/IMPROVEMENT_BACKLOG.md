# IMPROVEMENT_BACKLOG

> 用途：集中追蹤「改善 / 修復」事項，避免零散在 notes。  
> 分類：`auto`（可直接落地的小修） / `discussion`（需先討論再動手）。  
> 流程：先在此列出 → 視需要開 T 任務 → 完成後標記狀態。

欄位說明：
- `id`：例如 I-0001。
- `category`：`auto` | `discussion`。
- `description`：1–2 行摘要。
- `impact`：影響範圍或風險。
- `status`：`todo` | `doing` | `done` | `parked`。
- `related`：對應的 T 任務或 notes 連結。
- `notes`：補充（可留 RAW 連結）。
